# ハンズオン
## 最小構成で動かす
### home.nixにパッケージ追加
ここでは*nixfmt*を追加する
nixfmt:  Nix言語専用の公式コードフォーマッター
```nix
home.packages = with pkgs; [
	nixfmt
];
```
---
Nixでは、pkgs.nameと書くことでNixpkgsからパッケージのビルド情報を参照できる。
pkgsには参照するNixpkgsのブランチ・リビジョンの情報、ビルド対象のプラットフォーム情報(OS・CPUアーキテクチャ)などが含まれている。
**補足**
自動生成された `flake.nix` にて GitHub リポジトリやアーキテクチャ名が `pkgs` 変数に渡されています。

---
`with pkgs` について
`pkgs.` を何回も書く代わりに、`with` を利用すると楽に記述できる。
```nix
[ 
	pkgs.tool1
	pkgs.tool2
]
with pkgs; [
	tool1
	tool2
]
```
---
### 反映する
```nix
home-manager switch --flake .
```
### 動作確認
```nix
nixfmt --version
```

home.packages = with pkgs; [

## パッケージのインストール home.packages
`home.packages` にユーザー環境へ入れたいパッケージ名を記述する。
```nix
home.packages = with pkgs; [
	nixfmt
]
```
[NixOS Search](https://search.nixos.org/packages?channel=unstable)で利用可能なパッケージを検索できる。
詳細はここから復習できる[[管理可能なパッケージ]]
##  パッケージの設定ファイルの配置 home.file
```nix
home.packages = with pkgs; [
	git
];
# home.nixからの相対パスを指定
home.file = {
	".gitconfig".source = ./git/.gitconfig;
};
```
```tree
dotfiles/ 
 ├─ flake.nix
 └─ home-manager/
	 ├─ home.nix
	 └─ git/ 
		 └─ .gitconfig
```
## パッケージの設定ファイルの生成 programs.package
programs.packageでインストールと設定ファイルを構築・配置できます
**Git**
設定ファイルに以下を定義。 `~/.gitconfig`として配置される。
- `user.name`
- `user.email`
```nix
home,packages = [
	# Gitの記述は不要
]

programs.git = [
	enable = true; # ここでGitをインストールすると指定できる
	settings = {
		user = {
			name = "MyNixName";
			email = "MyEmail@example.com";
		};
	};
];
```
**補足**
`programs.git.settings`などはHome Manager独自関数
利用可能な関数は[Option Search](https://home-manager-options.extranix.com/)で検索
別の方法[home.file.name.text](https://zenn.dev/trifolium/articles/642043cbae5f21)で確認
## 環境変数の設定 home.sessionVariables
`home.sessionVariables`にて設定できる。
```home.nix
home.sessionVariables = {
	GREETING = "Hello Nix";
}
```
`home-manager switch`後、
シェルを読み込み直すと`GREETING`が設定されていることを確認できる。
```zsh
echo $GREETING
Hello Nix
```
## アップデート
### パッケージの更新
`flake.nix`があるディレクトリ(`~/work/dotfiles`)にて、以下のコマンドを実行
```zsh
nix flake update nixpkgs
```
`flake.lock`が更新され、パッケージのビルドレシピが最新になる。
その後、以下のコマンドを実行すると、パッケージが更新される。
```zsh
home-manager switch --flake .
```
### Home Manager本体の更新
`flake.nix` があるディレクトリ（`~/work/dotfiles`）で、以下のコマンドを実行。
```zsh
nix flake update home-manager
```
`flake.lock`が更新され、パッケージのビルドレシピが最新になる。
その後、以下のコマンドを実行すると、パッケージが更新される。
```zsh
home-manager switch --flake .
```
### 本体更新後の作業
Home Manager本体を更新した際は、リリースノートを確認。
リリースノートに`home.stateVersion`を更新する旨が記載されていた場合、`home.nix` にて指定された値に更新する。
[公式リファレンス リリースノート](https://nix-community.github.io/home-manager/release-notes.xhtml)
```zsh
# 変更する場合、必ずリリースノートを確認する 
home.stateVersion = "25.11";
```
**補足**
home.stateVersionについて
後方互換性を担保し、破壊的変更を防ぐための機能。
必ずリリースノートを見て、既存設定に影響がないかを確認する。
判断に迷う場合は、無理に変更せず古い値のままでOK
