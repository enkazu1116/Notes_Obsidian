#Nix #HomeManager 
# ホーム直下以外に設定ファイルを配置
`home.file`はホーム直下以外にもファイルを配置できる。
また、`xdg.configFile`を用いると、`~/.config`配下に置けるため便利。
```home.nix
{
	# ~/.gitconfigとして配置
	home.file.".gitconfig".source = ./git/.gitconfig;
	
	# ~/.config/git/configとして配置
	home.file.".config/git/config".source = ./git/.gitconfig;
	
	# XDG_CONFIG_HOMEで指定されたディレクトリに配置
	# デフォルト値: .config/
	xdg.configFile."git/config".source = ./git/.gitconfig;
}
```
上記3行のうちいずれか一つの行を使用する。
# 設定ファイルの書き込み禁止を回避する
`home.file.toolname.source`で`.gitconfig`等のシンボリックリンクを作成する場合、
書き込み禁止なファイルとして配置される。
**例**
`git config --global`で値を書き込めない状態で配置
```zsh
git config --global user.name "Nix"
error: could not lock config file /home/endoikki/.gitconfig: Permission denied
```
## 対策
`mkOutOfStoreSymlink`を用いると、書き込み可能な状態でシンボリックリンクが作成される。
```home.nix
home.file = {
	".gitconfig".source = 
		config.lib.file.mkOutOfStoreSymlink 
		"/Users/endouikki/work/dotfiles/home-manager/git/.gitconfig";
};
```
これにより、設定で変更できるようになる。
**補足**
`config.lib.file.mkOutOfStoreSymlink` の仕様で、**絶対パス**を指定する必要があります。
### 保守性向上させる場合
```home.nix
let
	inherit (config.lib.file) mkOutOfStoreSymlink;
	dotfilesDir = "/Users/endouikki/work/dotfiles/home-manager";
in
{
	home.file = {
		".gitconfig".source = 
		mkOutOfStoreSymlink "${dotfilesDir}/git/.gitconfig";
	};
};
```
`let-in`はNix言語における変数のスコープを定義するキーワード
`let`: 変数を宣言・初期化
`in`: letを利用する
`inherit`はコード記述を簡略化するために使用。
```nix
let 
	a = 1;
in 
{ 
	# 普通に書く場合 
	a = a; 
	# 省略して書くことができる 
	inherit a; 
}
```
inherit()と書く場合、以下のように使用できる。
```nix
let
  x = {
    a = 1;
  };
in
{
  # 普通に書く場合
  a = x.a;

  # 省略して書くことができる
  inherit (x) a;
}
```
Nixでは、`{<引数>}: <式>`という形式で関数を定義する。
つまり、先ほど作成したhome.nixファイル全体で1つの関数となる。
```nix
# config が引数
{ config, ... }:

# 以降が式
{
  home.file = ...
}
```
関数の式の中では、引数を参照できる。
そのため、以下のように`config`という変数を利用できる
```nix
{ config, ... }:
let
  hoge = config.lib....
```
# 設定ファイルの分割
home.nixにユーザー環境を定義すると、将来的にファイルが肥大化する。
設定ファイルを分割すると、管理が楽になる。
