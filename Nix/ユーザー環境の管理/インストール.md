#Nix #HomeManager
# dotfilesディレクトリの準備
## dotfilesとは
明確な定義はない。
一般的には、ホームディレクトリにある設定ファイルを意味する(~/.gitconfigなど)
```zsh
mkdir work
mkdir work/dotfiles
```
## 設定ファイル作成
```zsh
cd ~/work/dotfiles
nix run home-manager/master -- init .
```
**コマンド解説**
*nix run ...*
パッケージをインストールせずに実行できる
```zsh
# home-manager をインストールせずに実行 
nix run home-manager/master -- <command> 
# インストール済みの環境における下記コマンドと同義 
home-manager <command>
```
*home-manager init*
設定ファイルを自動生成できる
```zsh
home-manager init <path>
```
Home Managerのinitコマンドにより以下が生成される。
- dotfiles/flake.nix
- dotfiles/home.nix

これらをGit管理に加えること
```zsh
git init
git add flake.nix home.nix
git commit -m "generate Home Manager config file"
```

### addしなかった場合
Nixの使用により、flake.nixがGit追跡状態でない場合、今後の行程で以下のようなエラーが発生する。
```zsh
nix run home-manager/master -- switch --flake .
error: Path 'flake.nix' in the repository "/home/ryu/work/dotfiles" is not tracked by Git.
```
**補足**
より正確には Flakes という機能の仕様です。  
FlakesではGit 追跡されたファイルのみを利用します（通常の操作の場合）。  
そのため、`flake.nix` 本体や `flake.nix` 内で参照するファイルが Git 追跡されていないと、ファイルが見つからない！とエラーが発生します。

# インストール
```zsh
nix run home-manager/master -- switch --flake .
```
**解説**
*nix run*を使い、ユーザー環境にないhome-managerコマンドを利用している。
書き下すと、以下のようになる
```
home-manager switch --flake .
```
- *switch*コマンド
  Home Managerの設定ファイルを読み込み、ユーザー環境に反映。
- --flakeオプション
  参照する設定ファイルを指定できる。
- 未指定時
  *~/.config/home-manager/flake.nix*が参照される。
## 確認
```zsh
home-manager --version
```

## インストール方法の種類
導入方法は大きく分けて2つ

| 方法                | 概要                 |
| ----------------- | ------------------ |
| Standalone        | 単体でHome Managerを使う |
| NixOS module      | NixOSに組み込む         |
| nix-darwin module | nix-darwinに組み込む    |
## バージョン管理方法の種類
バージョン管理方法2つ
- nix-channel
  標準的な方法。
  ただし、バージョン情報をGit管理に反映させるのが手間。
- Flake
  ロックファイルでバージョンを固定
  これにより、Gitにてバージョン管理やロールバックが容易になる
