```tree
.

    └── env

                ├── backend.tf
```

## ルートモジュールの記述
- Terraformのバージョン指定
- 使用するプロバイダの情報の設定
- プロバイダの設定
- tfstateファイルの格納先(バックエンド)の設定
## terraform.tf
*terraformブロック*
Terraform本体のバージョンと使用するプロバイダの情報を記述
```terraform
[ローカル名]= {
	source = [プロバイダの識別子。URL, 名前空間、プロバイダの名前など]
	version = [バージョンの条件]
}
```

### Terraformのバージョン条件の指定
`required_version`によって、Terraform本体のバージョン満たす条件を指定できる。

*各種プロバイダのバージョン条件の指定*
*required_providersブロック*
使用するプロバイダとそのバージョンを指定する。
### .terraform.lock.hcl
```terraform
terraform init
```
これを実行する際、`.terraform.lock.hcl`というファイルが作成される。
プロバイダのバージョンなどを記録するためのもの。
このファイルがあることで、次回以降の`terraform init`の実行時に同じプロバイダが選択される

## provider.tf
*providerブロック*
プロバイダの設定を記述する
```terraform
provider "[プロバイダのローカル名]" {
	[設定項目のキー] = [値]
}
```
同一のプロバイダに対して`provider`ブロックを複数個記述
リソース、データソースの記述の際に、プロバイダ設定を選択できる

### エイリアスとデフォルト設定
`alias`の項目がないブロックはデフォルト設定になる。
`resource`ブロックや`dataブロック`で`provider`という引数を指定しない場合
デフォルトの設定が使われる。

*デフォルトのタグの設定 **default_tagsブロック***
デフォルトのタグの設定をする。
タグで付与できるリソース全てにこれらのタグが付与される。
リソースがTerraformで管理されていること、その時の`STAGE`, `MODULE`の情報をリソースに付与
## variables.tf
*variableブロック*
入力パラメーターを定義する。
- ステージ名に対しては、`stage`という名前の入力パラメータと型を定義
- `default`の引数の指定がない場合
  パラメータの指定: 必須
  指定されていない場合: エラー
## main.tf
*resourceブロック*
引数の値やその一部に`var.`から始まる入力パラメーターの値を指定
文字列の一部にパラメータの値を埋め込みたい時
→ 引用符の中で`${[パラメータ名]}`と記述する
**this** 
- 識別子によるリソースの説明が不要な場合
- リソースのモジュールの中でその種類のリソースが1つしかない場合
### 子モジュールを呼び出す
*moduleブロック*
子モジュールの呼び出しに使用する。
1つのラベルを伴い、そのラベルでモジュールの識別子を指定する。
同一のルートモジュール内で一意になるように命名する必要がある。
**引数**
`source`の指定が必須。
子モジュールのディレクトリのパスを指定する。
`子ブロック内のvariable`ブロックで定義された入力パラメータを列挙する
## outputs.tf
子モジュールの中のリソースの属性などを呼び出したモジュールから参照できるようにする
その属性値をoutputブロックに指定。

## tfstateファイルの格納先の設定
### tfstateファイル
どのリソースがTerrafromの管理下にあるのかを示す重要な情報が入っている。
Terraformの操作と同期している必要がある。
**注意**
tfstateファイルはローカルに置かず、リモートの1箇所で管理するべき
その設定を記述するのが、terraformブロック内に記述する`backendブロック`

### backendブロック
1つのラベルを伴い、そのラベルで用いるバックエンドの種類を指定する。
*格納方法*
tfstateファイルを格納するストレージは、環境ごとに分ける

*引数の値には名前付きの値は使用不可*
`dataブロック`, `variable`, `localsブロック`のデータを使用できない。
つまり実行時の入力パラメータやリソースの状態によって、動的に決められない

### ストレージについての推奨事項
- Terraformの管理下に置かないこと
  Terraformの管理ファイルをTerraformが管理する循環の関係が生まれてしまう
- バージョニングを有効にする
- ストレージへのアクションの許可を制限すること

## localsブロック
ローカル変数を定義できるブロック
ルートモジュールや子モジュールで繰り返し使う値を定義し、モジュール内から参照
