#Azure #terraform 
# 基本コマンド
## ルートモジュールの初期化
```terraform
terraform init
```
ルートディレクトリで初めてTerraformを使うときに、実行する

## 実行計画を確認する
```terraform
AWS_PROFILE=admin terraform plan
```
リソースの操作をする`terraform apply`を実行したときに以下が表示される
- 操作対象となるリソース
- その操作の内容を示した「実行計画」
この実行計画を確認し、自分が意図したリソースの操作になっているかを確認する。

## リソースをデプロイする
```terraform
AWS_PROFILE=admin terraform apply
```
コンフィグファイルで宣言されたリソースをクラウドプロバイダー上に構築するときは、以上のコマンドを実行する。
### tfstateファイル
リソースをデプロイすると、実行したディレクトリに`terraform.tfstate`というファイルが作成される。
#### terraform.tfstate
- コンフィグファイルに記述したリソース
- クラウドプロバイダーにデプロイされたリソース
上記二つを結びつける重要なファイル
このファイルに記述されているリソースが、Terraformで管理されているリソースになる
##### 複数人で管理するとき
それぞれの開発者から参照や更新できるよう、リモートストレージに保存する
